## 登录功能说明

### 登录流程
1. 用户在登录页面输入用户名和密码
2. 前端将密码进行MD5加密
3. 后端验证用户名和密码
4. 验证成功后生成JWT Token
5. 前端保存Token到localStorage
6. 跳转到工作台页面

### Token说明
- **有效期**: 24小时
- **存储位置**: localStorage
- **键名**: `auth-token`
- **用户信息**: `user-info`

### 路由守卫
- 未登录用户访问需要认证的页面会自动跳转到登录页
- 登录后用户访问登录页会自动跳转到工作台
- Token过期后需要重新登录

### 登出功能
点击右上角的登出按钮（LogOut图标）即可退出登录，会清除本地存储的Token和用户信息。

## API接口

### 登录
```
POST /api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}
```

### 获取当前用户信息
```
GET /api/auth/user-info
Authorization: Bearer {token}
```

### 验证Token
```
GET /api/auth/validate
Authorization: Bearer {token}
```

### 登出
```
POST /api/auth/logout
```

## 故障排除

### 登录失败提示"用户名或密码错误"
1. 检查数据库中是否存在admin用户
2. 确认密码已更新为MD5格式: `0192023a7bbd73250516f069df18b500`

### Token验证失败
1. 检查JWT配置是否正确加载
2. 确认Token未过期（24小时有效期）

### 跨域问题
如果遇到跨域问题，检查后端`CorsConfig`配置是否正确。

## 安全建议

1. **生产环境建议使用BCrypt加密**
   当前使用MD5是为了简化，生产环境建议使用BCrypt。

2. **定期更换JWT密钥**
   修改`application.yml`中的`jwt.secret`配置。

3. **设置合理的Token过期时间**
   根据业务需求调整`jwt.expiration`配置。

4. **启用HTTPS**
   生产环境务必使用HTTPS协议保护Token传输。
