# 炊大皇SRM系统 - 数据库配置说明

## 支持的数据库类型

本系统支持以下数据库：
- **MySQL** 8.0+
- **PostgreSQL** 12+

## 配置文件位置

数据库配置文件位于：
```
chuidahuang-srm-backend/src/main/resources/application.yml
```

## 快速切换数据库

### 方法一：修改配置文件

编辑 `application.yml` 文件，找到以下配置：

```yaml
spring:
  profiles:
    active: mysql  # 修改此处：mysql 或 postgresql
```

将 `active` 的值改为：
- `mysql` - 使用MySQL数据库
- `postgresql` - 使用PostgreSQL数据库

### 方法二：启动时指定参数

在启动应用时通过命令行参数指定：

**Windows:**
```bash
mvn spring-boot:run -Dspring-boot.run.profiles=mysql
# 或
mvn spring-boot:run -Dspring-boot.run.profiles=postgresql
```

**Linux/Mac:**
```bash
mvn spring-boot:run -Dspring.profiles.active=mysql
# 或
mvn spring-boot:run -Dspring.profiles.active=postgresql
```

## MySQL 配置

### 1. 配置信息

在 `application.yml` 的 `mysql` profile 部分：

```yaml
---
spring:
  config:
    activate:
      on-profile: mysql
  datasource:
    druid:
      driver-class-name: com.mysql.cj.jdbc.Driver
      url: jdbc:mysql://主机:端口/数据库名?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
      username: 用户名
      password: 密码
```

### 2. 初始化数据库

执行MySQL初始化脚本：
```bash
mysql -u root -p < init-database.sql
```

或使用MySQL客户端执行脚本内容。

### 3. 配置示例

```yaml
url: jdbc:mysql://10.220.5.164:3306/cdh_test?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
username: srmtest
password: SRMhw723SIIg
```

## PostgreSQL 配置

### 1. 配置信息

在 `application.yml` 的 `postgresql` profile 部分：

```yaml
---
spring:
  config:
    activate:
      on-profile: postgresql
  datasource:
    druid:
      driver-class-name: org.postgresql.Driver
      url: jdbc:postgresql://主机:端口/数据库名
      username: 用户名
      password: 密码
```

### 2. 初始化数据库

执行PostgreSQL初始化脚本：
```bash
psql -U postgres < init-database-postgresql.sql
```

或使用psql客户端：
```bash
psql -U postgres
\i init-database-postgresql.sql
```

### 3. 配置示例

```yaml
url: jdbc:postgresql://localhost:5432/cdh_test
username: postgres
password: your_password_here
```

## 注意事项

### 1. 数据库驱动

系统已同时包含MySQL和PostgreSQL的JDBC驱动，无需额外安装。

### 2. 主键策略

实体类使用 `IdType.AUTO`，会根据数据库类型自动选择：
- MySQL: 使用 `AUTO_INCREMENT`
- PostgreSQL: 使用 `SERIAL` (序列)

### 3. 分页方言

系统会自动检测数据库类型并配置正确的分页方言，无需手动配置。

### 4. SQL兼容性

本系统使用的SQL语法已做兼容处理，两种数据库均可正常使用。

## 验证配置

启动应用后，可以检查日志确认数据库连接成功：

```

```

或访问 Druid 监控页面：
```
http://localhost:8080/api/druid/
用户名: admin
密码: admin123
```

## 常见问题

### Q1: 切换数据库后启动失败？
**A:** 检查以下几点：
1. 数据库是否已创建
2. 用户名密码是否正确
3. 数据库服务是否启动
4. 防火墙是否允许连接

### Q2: 如何确认当前使用的数据库？
**A:** 查看启动日志中的数据库连接信息，或访问Druid监控页面查看。

### Q3: 可以同时连接多个数据库吗？
**A:** 当前配置每次只能使用一种数据库。如需多数据源，需要进行额外的多数据源配置开发。

## 数据库连接信息参考

| 配置项 | MySQL | PostgreSQL |
|--------|-------|------------|
| 默认端口 | 3306 | 5432 |
| JDBC URL格式 | `jdbc:mysql://host:port/db` | `jdbc:postgresql://host:port/db` |
| 驱动类名 | `com.mysql.cj.jdbc.Driver` | `org.postgresql.Driver` |
| 自增主键 | `AUTO_INCREMENT` | `SERIAL` |
| 注释符号 | `--` 或 `#` | `--` |
| 字符串引号 | 单引号 `'` | 单引号 `'` |
